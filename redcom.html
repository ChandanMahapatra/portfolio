Overview

The Sigma Client is a Windows application that allows secure communication among its users. A usability study was conducted to see if users could easily understand and use the key features of the application.
Please Note: Study material has been heavily redacted due to client confidentiality.

Tools

Problem

REDCOM has designed and developed a new Sigma Client Application for the Windows operating system. Since it is a new communication application, it has not been tested to see if users can successful perform various important communications tasks using the application.

Solution

The main purpose of this usability study is to recognize important usability issues with the product and to determine if all the components of the application were functioning well. This is important, as glitches can prevent the application from working properly and the users might face challenges while performing various tasks.

Process

Heuristic Evaluation

We conducted a preliminary heuristic evaluation based on Don Norman’s 10 Usability Heuristics for User Interface Design to analyze the interface from different angles and to identify key design problems.

Research Questions

We developed research questions based on the heuristic evaluation we conducted. The research questions also accounted for client concerns and our primary goals.

Don Norman’s 10 Usability Heuristics for User Interface Design

Participant Requirement

The usability test had ten RIT students who had some experience with voice and video applications. These students were recruited via flyers designed by us. Each potential participant filled out a screener questionnaire.

Eligible participants were contacted through email to schedule the study. All the participants were given a gift card as compensation for their participation.

Test Plan

All the participant had to perform five tasks with the Sigma Client application. Each task in the study was based on a scenario. Quantitative data was collected during the test using 5 point Likert Scale and SUS questionnaires. Qualitative data was collected by using post-task questions and debriefing sessions.

We used the system usability scale, which is an industry standard tool, for measuring the usability of the system.

Location and Setup

The usability tests were carried out in the HCI usability testing lab at B. Thomas Golisano College of Computing and Information Sciences building, RIT.

The lab has two rooms; one is the testing room which has a one-way mirror and the other room is the observation room. The usability test with the participants was conducted in the testing room.

We set up one webcam and one microphone to record the participant in the testing room.The usability testing lab was equipped with two Morae Systems (Morae recorder and Morae observer).

Final Report and Client Presentation

  • We created a final report that contained our findings from the heuristic evaluation and usability test.
  • This report helped identify the key problems and undocumented bugs in the application’s functionality.
  • We also provided design and workflow recommendations that could help improve the usability of the application.
